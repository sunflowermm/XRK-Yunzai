## 已废弃表述（文档中不得再出现）

| 废弃 | 替换为 |
|------|--------|
| `streams/` 工作流目录 | 仅 `stream/` |
| `stream/device.js` 设备语音工作流 | 已删除；设备走 `http/device.js` Event WS + `chat` 工作流 |
| ASR/TTS 工厂、`volcengine_asr`/`volcengine_tts` | 已移除；仅保留 LLM 工厂 |
| WS 下行 `asr_interim`/`play_tts_audio` | `reply` / `typing` / `error` / 心跳响应 |
| Web 控制台「语音模式」 | Event / AI 双模式 |
| `global.mcpServer` | `StreamLoader.mcpServer` 或 `MCPToolAdapter.getMCPServer()` |
| `aistream.mcp.remote.servers` + `selected` | `aistream.mcp.remote.mcpServers` JSON 块数组 |
| `cfg?.aistream` 散落（`lib/` 内） | `getAistreamConfigOptional()` |
| LLM 运营商手写 Input | 控制台动态 **Select**（`LLMFactory.listProviders()`） |
| `mcp.defaultStreams` / `defaultRemoteMcp` / `toolMergeStrategy` | 已移除；进阶 MCP 在 [XRK-AGT](https://github.com/sunflowermm/XRK-AGT) |
| 根 README Roadmap 勾选清单 | 演进方向指向 XRK-AGT，不在本仓维护 |
| 删除 `lib/plugins/config.js` | TRSS 插件（zmd-plugin 等）依赖 `makeConfig()`，须保留 |

自动化：`tests/framework/doc-consistency.test.mjs` 扫描部分关键文件。

## 审查步骤

1. 跑 `pnpm test`，确认基准与配置三件套通过
2. 跑 `node scripts/validate-skills.mjs`
3. 改模块数量或配置段时，grep 全文：`streams/`、`ASR`、`TTS`、`global.mcpServer`、`remote.servers`
4. 更新 [docs/README.md](README.md) 索引（新增文档时）
5. 更新 [.cursor/skills/SKILL_INDEX.md](../.cursor/skills/SKILL_INDEX.md)

## 与 XRK-AGT 差异（勿照搬路径）

| AGT | Yunzai |
|-----|--------|
| `core/system-Core/` | `plugins/system-plugin/` |
| `src/infrastructure/` | `lib/` |
| `docs/coding-style.md` Node 26 表 | Node 24+，见 Yunzai 版 coding-style |
| Tasker 层文档 | 适配器在 `adapter/`，无独立 Tasker 文档层 |
| 长期 Roadmap / 智能体平台特性 | [XRK-AGT](https://github.com/sunflowermm/XRK-AGT) |

对齐的是**方法论**（基准测试、配置三件套、skill 索引），不是目录字面路径，也不是未接入 Yunzai 的规划项。
