# 框架测试指南

> 对齐 XRK-AGT 的框架基准测试思路，适配 Yunzai 目录模型（`plugins/system-plugin/` + `lib/`）。

---

## 命令

| 命令 | 说明 |
|------|------|
| `pnpm test` | 运行全部框架单测（`tests/framework/*.test.mjs`） |
| `pnpm test:fast` | 同 `pnpm test`（预留与集成/E2E 拆分） |
| `pnpm lint` | ESLint：`lib/`、`plugins/`、`app.js`、`start.js` |
| `node scripts/validate-skills.mjs` | 校验 `.cursor/skills/*/SKILL.md` frontmatter |

环境变量：`XRK_TEST=1`（测试引导可选使用）。

---

## 目录

```
tests/
├── framework/
│   ├── config-alignment.test.mjs    # 默认 yaml ↔ system.js schema ↔ 常量列表
│   ├── config-constants.test.mjs    # 配置分类与工厂后缀
│   ├── module-inventory.test.mjs    # system-plugin 模块数量基准
│   ├── plugin-dir-scanner.test.mjs  # 工作流仅扫 stream/
│   ├── lib-conventions.test.mjs          # lib/ 约定：fs、关闭链、Bot.makeLog
│   ├── lib-tools.test.mjs                # path-guards、InputValidator、formatBytes 等
│   ├── system-plugin-conventions.test.mjs  # system-plugin 后端约定
│   └── doc-consistency.test.mjs     # 文档禁止 streams/、ASR/TTS 残留表述
└── helpers/
    └── system-plugin-baseline.mjs   # SYSTEM_PLUGIN_BASELINE 标准值
```

## system-plugin 基准（须与代码一致）

维护位置：`tests/helpers/system-plugin-baseline.mjs`

| 子目录 | 基准数量 | 说明 |
|--------|----------|------|
| `http/` | 10 | HTTP API 模块 |
| `stream/` | 7 | AI 工作流（chat/memory/tools/database/desktop/web/browser；无 `device.js` 语音流） |
| `plugin/` | 12 | 内置插件 |
| `events/` | 5 | 事件监听 |
| `adapter/` | 5 | 协议适配器 |

增删内置模块时：**先改基准常量，再改文档**（`SYSTEM-PLUGIN.md`、`文档审查清单.md`）。

---

## 配置三件套

1. `config/default_config/{name}.yaml` 存在（`GLOBAL_CONFIG_NAMES` + `PORT_CONFIG_NAMES`）
2. `plugins/system-plugin/commonconfig/system.js` schema 含对应段
3. `lib/config/config-constants.js` 列表与上述一致

工厂 LLM yaml（`*_llm.yaml`）由 Factory 读取，**不**列入 `GLOBAL_CONFIG_NAMES` 三件套。
